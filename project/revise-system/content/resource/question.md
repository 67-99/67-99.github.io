# 题库文件格式说明

## 一、文件位置与命名
- 题目集文件位于程序同级的 `questions/` 文件夹内
- 文件格式：JSON，编码 UTF-8
- 文件名示例：`高等数学.json`、`默认题库.json`
- 每个 JSON 文件包含一个题目集，根对象必须包含 `"questions"` 数组

## 二、整体结构
{
  "questions": [
    { 题目对象1 },
    { 题目对象2 },
    ...
  ]
}

## 三、题目对象（Question）的通用字段
| 字段           | 类型           | 必填 | 说明                                                                             |
|----------------|----------------|------|---------------------------------------------------------------------------------|
| id             | int 或 string  | 是   | 题目标识，可以是数字或非空字符串（如 "5a"），允许重复                               |
| type           | string         | 是   | 题型：`"single"`（单选）、`"multiple"`（多选）、`"fill"`（填空）、`"essay"`（简答） |
| question       | string         | 是   | 题目内容，可包含行内 LaTeX 公式（用 `$...$` 包裹）                                 |
| answer         | 根据题型而定    | 是   | 正确答案（详见各题型说明）                                                        |
| explanation    | string         | 否   | 全局解析，可包含 LaTeX                                                           |

## 四、各题型详细格式

1. 单选题（type = "single"）
额外必填字段：
- options: string[] —— 选项列表，每个元素为选项文本（支持 LaTeX）
- option_explanations: string[]（可选） —— 与 options 一一对应的选项解释，提交后显示

answer 格式：
- 一个字符串，内容必须与 options 中某一项完全匹配

示例：
```json
{
  "id": 1,
  "type": "single",
  "question": "函数 $f(x)=\\begin{cases}x^2, & x>0\\\\ -x, & x\\leq 0\\end{cases}$ 在 $x=0$ 处是否连续？",
  "options": ["连续", "不连续", "无法判断"],
  "answer": "不连续",
  "explanation": "左右极限不相等。",
  "option_explanations": ["", "正确，左极限为0，右极限为0但函数值为0，实际连续？请核对。", ""]
}
```

2. 多选题（type = "multiple"）
额外必填字段：
- options: string[] —— 选项列表

answer 格式：
- 字符串数组，元素为正确选项的文本，顺序无所谓，程序会比较排序后的列表

示例：
```json
{
  "id": 2,
  "type": "multiple",
  "question": "以下哪些是 Python 的不可变类型？",
  "options": ["int", "list", "tuple", "dict", "str"],
  "answer": ["int", "tuple", "str"],
  "explanation": "list 和 dict 是可变的。"
}
```

3. 填空题（type = "fill"）
无 options 字段，用户直接在输入框中填写答案。

answer 格式：
- 字符串：普通文本答案，比较时忽略大小写
- 字符串数组：用户输入与数组中任意一项匹配即算正确
  数组中某项若以 `"regex:"` 开头，则其后部分视为正则表达式，进行完全匹配（fullmatch），例如 `"regex:\\s*3\\.14\\s*"`
  非 `regex:` 的项仍然按忽略大小写的普通字符串匹配

示例1（单个答案）：
```json
{
  "id": 3,
  "type": "fill",
  "question": "Python 中定义函数的关键字是 ____。",
  "answer": "def"
}
```

示例2（多个可接受答案，含正则）：
```json
{
  "id": 4,
  "type": "fill",
  "question": "圆的周长公式中，常数 $\\pi$ 的近似值（两位小数）。",
  "answer": ["3.14", "regex:\\s*3\\.14\\s*"]
}
```

4. 简答题（type = "essay"）
无 options 字段，用户输入多行文本。

answer 格式：
- 字符串：参考答案，仅用作展示对比，不自动判分

示例：
```json
{
  "id": 5,
  "type": "essay",
  "question": "请简述列表和元组的区别。",
  "answer": "列表可变，元组不可变；列表用方括号，元组用圆括号。",
  "explanation": "回答要点：可变性、语法符号。"
}
```

## 五、题目内容中的 Markdown 支持（含表格、代码块、Mermaid）

在 `question`（题目描述）和 `options`（选项文本）中，除了使用 `$...$` 包裹 LaTeX 公式外，还可以使用以下 Markdown 语法，程序会自动渲染为对应格式（渲染失败时会有备选显示）。

| Markdown 语法 | 说明 |
| -------------------------------- | ---------------------------------------------------------------------- |
| `` `代码` `` | 行内代码，显示为等宽字体，不会处理其中的 LaTeX |
| ```` ```语言标签 ... ``` ```` | 代码块，支持多种语言标签；若标签为 `mermaid`，则尝试渲染为 Mermaid 图表（见下文） |
| `\| 列1 \| 列2 \|`<br>`\| --- \| --- \|`<br>`\| 数据1 \| 数据2 \|` | 标准 Markdown 表格（支持对齐符号 `:---`、`:---:`、`---:`），渲染为带边框的 HTML 表格 |

**注意事项**：
- 表格必须包含表头行、分隔行（含 `---`）和数据行，每行都以 `|` 开头和结尾。
- 表格内的单元格内容可包含 LaTeX 公式（用 `$...$` 包裹），程序会正常渲染。
- 代码块中的内容**不会**进行 LaTeX 或 Markdown 解析，原样显示（Mermaid 除外）。

### Mermaid 图表支持

当代码块的语言标签为 `mermaid` 时，程序会尝试将内容渲染为 SVG 图表。若本地渲染库不可用，会自动切换为备选方案：
- 显示一个指向 `mermaid.ink` 的图片链接（可点击查看）。
- 同时提供直达 [Mermaid Live 编辑器](https://mermaid.live/) 的链接，方便在线编辑和导出。
- 原始代码块也会原样显示，便于手动复制。

**示例**（在 JSON 中需转义换行和反引号）：
```json
{
  "id": 6,
  "type": "single",
  "question": "根据下图选择正确的描述：\n```mermaid\ngraph TD\n A[开始] --> B{判断}\n B -->|是| C[处理]\n B -->|否| D[结束]\n```",
  "options": ["流程包含分支", "流程无循环"],
  "answer": "流程包含分支"
}
```

### 行内代码与表格示例

```json
{
  "id": 7,
  "type": "single",
  "question": "以下代码 `print('Hello')` 的输出是什么？\n\n| 语言 | 输出 |\n| ------ | --------- |\n| Python | `Hello` |\n| Java | 编译错误 |",
  "options": ["`Hello`", "编译错误"],
  "answer": "`Hello`"
}
```

## 六、题目内容中的 LaTeX 支持
- 使用 `$...$` 行内公式，可包含常见的 LaTeX 命令（详见程序内置符号表）
- 支持 `\begin{cases}...\end{cases}` 环境，程序会渲染为带大括号的多行排版
- `\frac`、`\sqrt`、`\sum`、`\int`、希腊字母等均可使用
- 若渲染失败，按钮或标签会自动降级为可读 Unicode（如 `\cup` → ∪）
- 文本中勿忘转义 JSON 特殊字符，比如 `\\` 表示 LaTeX 的换行 `\\`，因此 JSON 中应为 `\\\\`
  - 示例：`"\\begin{cases}x^2, & x>0\\\\ -x, & x\\leq 0\\end{cases}"` 在 JSON 字符串中需写为 `"\\begin{cases}x^2, & x>0\\\\ -x, & x\\leq 0\\end{cases}"`（注意双重转义）

## 七、注意事项
- JSON 必须为合法格式，建议使用专门的编辑器或 Python 的 json 模块校验
- 题目 id 可以是数字或字符串（如 "5a"），不必连续或唯一；但建议使用有意义的标识，以便在显示时便于区分。
- 程序会为每个题目自动生成内部索引 `_index`（从 0 开始按数组顺序），该索引仅供内部使用（缓存、定位），无需在 JSON 文件中提供。
- 单选题和多选题的选项支持 LaTeX 渲染，若渲染失败会显示 Unicode 降级文本
- 填空题和简答题的 answer 字段中不要使用 `$...$`（无需渲染，仅作字符串比较或展示）
- 所有文本均可包含换行符 `\n`，程序会自动转换为 `<br>`
- 在 JSON 中书写 Markdown 表格或代码块时，请注意转义换行符（`\n`）和反斜杠，确保 JSON 格式合法。例如代码块的三反引号需写作 `\`\`\``（在 JSON 字符串中为 `"```"`，但通常可以直接写为多行字符串，Python 解析时会保留换行）。
